# Fork Ref
不是 react 正式的 hook，主要功用是把兩個 ref 整理成一個。當擔心其中一個 ref 是 empty 的時候會，會做另一個 ref 補上

## 使用情境
情境滿特殊的，今天有個 component `SimpleProductCard`，其中已經有一個 ref: `cardRef`，以及與其搭配的 hook
但今天使用該 component 的 parent 和他是不同的 project，我們希望不要更動太多 `SimpleProductCard` 的內容，可以從 parent 額外送入 ref 來使用，並把 hook 設置在 parent 中
因此我們在 `SimpleProductCard` 加上了 `forwardRef` 來對外開放他的 ref，大致如下
```javascript
const SimpleProductCard = React.forwardRef((props, ref) => {
	const cardRef = ref ? ref : useRef(); // illegal

	useEffect(() => {
		// do something with cardRef
	}, [])

	return <div ref={cardRef}>Card</div>
});

SimpleProductCard.displayName = "SimpleProductCard";
```
```javascript
const SponsoredProduct = ({ products }) => {
	const prodRef = useRef();

	useEffect(() => {
		// do something with prodRef
	}, [])

	return <SimpleProductCard ref={prodRef} />
}
```
可以看到 `SponsoredProduct` 就只是簡單回傳 `SimpleProductCard`，但會出現這樣的架構其實是因為兩者處在不同 project 中，我們希望維持 `SimpleProductCard` 的樣子，但在 `SponsoredProduct` 中多做點其他事
然而 `SimpleProductCard` 會在許多地方被用到，不一定個地方呼叫時都會再給 ref，所以我們放了個條件式 `const cardRef = ref ? ref : useRef();` 來確保 `cardRef` 不會為空
但 hook 的使用上**不可以**被放在條件式中！因此這裡的寫法是有問題的
這時候就能改成上面提到的 folkRef 作法
```javascript
const cardEl = useRef()
const cardWithForkEl = useForkRef(ref, cardEl);
```
