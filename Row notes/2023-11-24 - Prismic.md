# Prismic
## About Prismic
Prismic [readme](https://github.com/Houzz/prismic-cms/#readme)
Prismic 是一個 CMS （內容管理系統），簡單描述他的系統分配：
- Type: 算是一個我們定義出來的模板，例如 `Mp Guide Page`，其中包含可能會用到的各種區塊(slice)，以及slice中的data格式選項等
- Doc: 根據 Type 衍生出來的頁面內容，比方說基於 type: `Mp Guide Page`，我們可以做出頁面 **How To Choose a Bathroom Vanity: Your Step-by-Step Guide** 的內容，甚至是其他同類型的頁面，每個頁面對應到一份 (eg: uid: `how-to-choose-bathroom-vanity-buying-guide` )。而同一份 doc 中還能規劃對應的 lang 內容
一般來說，我們是使用 prismic-cms，service 中包含了 graphql, apps
## How to develop
### Prismic CMS
Prismic CMS Local Setup [Doc](https://docs.google.com/document/d/1b5s__yq3ttWN1XVpF5MG0Ytlez7FbUmcadNJWCeI78c/edit?usp=sharing)
### Jukwaa
Steps:
1. 在 staging prismic 上測試 doc field
2. 在 prismic gql playground 上測試 gql schema
3. 確認可以拿到 data 後，再到 jukwaa 測試 gql
4. setup data fetcher

local fetch prismic staging 的 data 要先加這兩個 cookie
1. [https://www.stghouzz.com/ad/test/kv=usePrismicApiGql:false](https://www.stghouzz.com/ad/test/kv=usePrismicApiGql:false)
2. [https://www.stghouzz.com/ad/test/kv=prismicEnv:staging](https://www.stghouzz.com/ad/test/kv=prismicEnv:staging)
### Create new Type and Doc
詳細可以參考 [PR](https://github.com/Houzz/prismic-cms/pull/1157)
1. 規劃 type 的名稱
2. 設定 data fetcher 以及對應的 gql schema，這是 graphql 可以拿到資料的關鍵，會照著 class `PrismicDocument` 的需求去設定，包含
	1. getDocumentPrimaryPart
	2. getDocumentBodies
	3. getAllDocumentPrimaryPart
	4. getAllDocumentBodies
3. 順便設定基本的 slice type
### Create new slice
詳細可以參考 [PR](https://github.com/Houzz/prismic-cms/pull/605/files#diff-e10dac9de9b6b5abb0828bd57b378f570231be30d8fa5062c0590fa37fbe9e0c) （其實這是透過 command `npm run build-fragment` 生成的）
做法一：
1. 從 staging prismic.io 增加新的 slice 到 type，或是修改現有的 slice
2. 執行 `npm run build-fragment` ，他會更新 slice schema 到 local 對應的地方。<br>像是 `getDocumentBodies` 中的 `FragmentBody`（這裡為例就是 MPCategoryFragmentBodyPart1.gql），讓 graphql 可以產出對應格式的資料
3. 後續 graphql 在執行的時候，會自己去抓對應的環境中的 .gql file，以拿到正確的格式
做法二（[Readme](https://github.com/Houzz/prismic-cms?tab=readme-ov-file#how-to-create-a-new-slice) 建議）：
1. 在 slice library 開一個新的 tab, eg, ‘Media Object’
2. 設定 usage type
3. 打開 slice zone （但目前沒辦法透過 ui，需要從旁邊的 JSON editor 手動補，我先複製了隔壁的部分設定，詳細 fields 在用拖拉的加入），其中 usage 跟 body 的編號要往後推
	```json
"Contest Vote Card Group" : {
    "usage52" : {
      "type" : "Group",
      "config" : {
        "fields" : {
          "type" : {
            "type" : "Text",
            "config" : {
              "label" : "type"
            }
          }
        },
        "label" : "Usage"
      }
    },
    "body52" : {
      "type" : "Slices",
      "fieldset" : "Slice zone",
      "config" : {
        "labels" : { },
        "choices" : {
          "contest_vote_card_group" : {
            "type" : "Slice",
            "fieldset" : "Contest Vote Card Group",
            "description" : "A list of Vote Card for the contest page",
            "icon" : "chrome_reader_mode",
            "display" : "list",
            "non-repeat" : {
              "tracking_id" : {
                "type" : "Text",
                "config" : {
                  "label" : "Tracking ID. You can specify a unique tracking id used for this slice. If not present, it will use the title. The format for this field should be \"xxx-xxx-xxx\"."
                }
              },
              "anchor_id" : {
                "type" : "Text",
                "config" : {
                  "label" : "Anchor ID. You can specify a unique anchor id used for jumping to this section of the page. White space is removed and characters made lowercase."
                }
              }
            }
          }
        }
      }
    }
	```
4. 把 JSON data 複製到 [SliceFields](https://github.com/Houzz/prismic-cms/blob/master/apps/cms/data/SliceFields.json)
	1. 但好像不更新也沒差，`npm run build-fragment` 就會自動更新 `SliceMap` 惹
5. 更新 [slice query](https://github.com/Houzz/prismic-cms/blob/master/packages/prismic/src/data-fetchers/gql/GetSliceLibrary.gql)
6. 再把 component 新增到 [Slice.jsx](https://github.com/Houzz/prismic-cms/blob/master/packages/prismic/src/templates/slice-library/Slice.jsx)

## Troubleshooting
1. 發生 `no service name specified` 的 error message
	1. 有可能是因為 GrowthBook 沒連上，檢查 error message `GrowthBook: feat eval service is not available`
	2. 在 `config-override/system/dev.js` 加入 `featEvalThriftSocket: 'feat-eval-service.backend.stg-main-eks.stghouzz.com:9097'`
