# Create UI annotation
<unknown url="https://app.notion.com/p/fc3ed12182f7476cbd9b5d3f299f0e4e#3420748884ad48c08eb28f69f5e3ac63" alt="bookmark"/>
L1 section, L2 container, L3 component的命名，都是定義好在[all_attributes.json](https://github.com/Houzz/omnilog/blob/master/definitions/all_attributes.json) 裡面的，因此在要新增名字的時候，都要開一個相關PR來新增
新增完名稱後，同樣是需要跑 `./build.sh` ，有趣的是，script會檢查再 json 中名稱的擺放順序
# Naming tips
1. Need `hz-track-me` to enable **click** tracking
2. Should no impressions on `Sections`, because `Section` is too big
3. container should be on the `<ul>` groups. having a `section` and `container` be the same element doesn't provide us anything
4. on `Section` name, we call it "sections" but the actual definitions don't need to say "section":<br>[https://github.com/Houzz/omnilog/blob/master/definitions/all_attributes.json#L486](https://github.com/Houzz/omnilog/blob/master/definitions/all_attributes.json#L486)
5. on `Component` name, "Item" doesn't convey anything. we can also suffix "Card".
6. use "List" instead of "Items” on `Section` name, like `Category List`
7. `UIEvents.ELEMENT_CLICKED` should follow with `data-cta`
8. Impression of Recommendation Carousel: [https://docs.google.com/presentation/d/1lO0ROPk4-94RF5z6uGrs2ylfxa-uKqyjwIjqqqhloPI/edit#slide=id.g18af509b684_0_54](https://docs.google.com/presentation/d/1lO0ROPk4-94RF5z6uGrs2ylfxa-uKqyjwIjqqqhloPI/edit#slide=id.g18af509b684_0_54)
9. container name 跟 component name 有一定關聯，這樣在Amplitude可以直接搜尋L3，如：
	- container: Product Cateogry
	- component: Product Category Card
