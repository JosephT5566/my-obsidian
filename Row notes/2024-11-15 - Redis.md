# Check Redis on staging
要先安裝 redis-cli: `brew install redis`
1. 直接在 local ter 下 cmd `redis-cli -h 192.168.10.74 -p 7000 -c`
2. 這樣就連上 redis 了，再繼續下 cmd 查詢 `get relatedQueries:101-products-table_lamps` （查詢 key 為 `relatedQueries:101-products-table_lamps` 的值）
# Check Redis on Prod
<callout icon="🚨" color="gray_bg">
	Prod 上僅能查看
</callout>
這就要透過 rq pods 了，在 stg 用的 cmd 一樣，只是需要在 pod 的 ter 下
1. 到 prod 的 [rq worker](https://rq-aws.houzz.net/)，這裡我們進到 ads_batch 的 [container](https://rq-aws.houzz.net/ads_batch)
2. 一樣 Debug pod button → create debug pod
3. 接著就會看到自己開的 debug pod name，可以點擊 copy 複製 cmd<br>`kubectl exec -it rq-ads-batch-2024-11-14-joseph.tseng-pkwq8 -n backend -c rq-worker-container -- bash`

從 k8s 連上 debug pod
1. 在本機 ter 用 `tsh kube ls` 看看有哪些 clusters （順便 login）
2. 選擇 `prod-batch` 這個 cluster: `tsh kube login prod-batch`
3. 下 cmd `kubectl exec -it [pod-name] -n backend -c rq-worker-container -- bash`<br>（也就是剛剛複製的 `kubectl exec -it rq-ads-batch-2024-11-14-joseph.tseng-pkwq8 -n backend -c rq-worker-container -- bash`）
4. 這樣就進到 pod 的 ter 了，接著我們連到 redis `redis-cli -c -h 10.1.3.151 -p 7000`
	1. or `redis-cli -c -h 10.1.1.106 -p 7000`
5. 下 query 查詢 `get relatedQueries:101-products-table_lamps`
