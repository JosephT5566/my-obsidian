# **What’s thrift**
Facebook 的開源 RPC 框架。主要用於各個服務之間的 RPC 通信，支持跨語言，常用的語言比如C++, Java, Python, PHP, …
RPC: Remote Procedure Call. 去call位在別的server的procedure (而不是在本機端的、單體的)

**Thrift vs. GraphQL:**
- GraphQL 是給 FE 使用的 API Query Layer
- Thrift 是 Service-to-Service 的 RPC Layer
- GraphQL Resolver 內部會透過 Thrift 呼叫後端服務取得資料
# C2 thrift service
How to develop a C2 Service
1. **Define thrift**
	1. Modify in [Houzz/c2thrift/](https://github.com/Houzz/c2thrift/), the single source of truth of our service protocol (sample [PR](https://github.com/Houzz/c2thrift/pull/34/files))
	2. Change c2thrift git submodule pointer at C2
2. Run thrift gen and class gen
	1. Git hook or ./c2/ <br>在切換 branch 時，c2thrift branch 的指標也會改變，這時候 git hook 就會同時去 gen 出對應的 thrift
	2. 以 c2 來說，執行 `c2/service/generateC2Thrift.sh` 就會生成 `c2/service/thrift/…/xxx.php` 的檔案，這些在後續我們要實作 php code 時就能直接調用（php IDE 就能直接看到預期的 class/object）
3. Implement php code (business logic)
	1. Modify in [Houzz/c2/services/handlers](https://github.com/Houzz/c2/tree/master/services/handlers)

# Connect to thrift
- thrift service 在用 curl connect 時，不會收到 response
	- 針對非 load balancer 的 service，我們可以透過 `kubectl port-forward svc/threed-image-classification-service 9090:443 -n backend` 讓 [localhost](http://localhost) 127.0.0.1 的 port 9090 直接連到對應的 service （需要 login `aws sso login`）
	- 接著再下 `curl -v `[`http://localhost:9090`](http://localhost:9090/)，成功連線時，最終會出現 Request completely sent off 然後就會 hang 住
- Thrift service
