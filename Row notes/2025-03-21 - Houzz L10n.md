# Lang file
lang file 中 `_author` 跟 `_id` 是 **required** ([ref](https://engwiki.houzz.tools/doc/lang-hgt-and-hgttag-7XRrioeApG#h-id-mandatory))
其中 `_id` 的生成是從 Webpack 中執行，邏輯是 filepath + timestamp 的 MD5 hash ([code](https://github.com/Houzz/jukwaa-core/blob/76a53e7696ac7001addaca7f5b575bde8496b795/src/webpack/loaders/lang-loader/index.js#L67))
```javascript
var ts = Date.now();
var id = cryptoUtil.createMD5Hash(filePath + ts);
```
有兩種做法可以生成 `_id`
1. 將上述的 code 丟到 AI 去生，filePath = "src/product-reviews-summary-dropdown/productReviewsDropdown.lang"
2. 將新的 lang file （不需要有 `_id` key） 放到 jukwaa 任何一個 apps/components 中，並且被現有的 js file imported。然後執行 `NODE_ENV=dev npm run build-apps -- -a <app-name>` ，其中的 `build-apps` 就會跑 webpack
