# About this My Wiki
這是我的知識資料庫，你的任務是協助我把零散的資料整理成一套彼此連結、好查找的筆記
## 資料夾與文件功能說明
- Row notes：用來放置原始的歷史資料素材，你只能讀取，不能修改這裡面的內容。
- My Wiki：由你來建立、更新和整理的資料夾，採用扁平式資料夾結構 (知識庫是依靠「Index.md」和[[雙向連結]]來組織)。
- Templates：存放 Obsidian 筆記範本。這個資料夾不是知識頁面，不需列入「Index.md」。
	- Concept：技術、協定、設計模式或術語的概念說明。
	- How-To：可重複執行的操作流程。
	- Troubleshooting：錯誤、故障與事件的調查及解決記錄。
	- Code Snippet：可重用的程式碼、指令或查詢。
	- Tool or Library：工具、framework 或 library 的使用與評估。
	- Decision Record：保留技術決策的背景、選項與後果。
	- Learning Note：文章、課程、talk 或實驗的學習摘要。

- Index.md：這個知識庫的目錄大綱，用來列出知識庫中的所有頁面以方便 AI 檢索。
	- Index 內的每個項目都是 My Wiki 中的一個頁面，包含了：連結 + 一句話說明。
	- 範例：Cache — 一個用來加快讀取速度的中間層
	
- Dialog.md：每當知識庫做了任何更新，就在日誌最後面添加一行。
	- 記錄的格式為：## [日期] 動作 | 說明。
	- 範例：[2026-06-30] 匯入文章 | What is Cache
	- 原始資料完成匯入後，請詳細登記每個匯入文件的名稱，以避免日後重複匯入。

- Import Registry.md：記錄 Row notes 的目前匯入狀態，供匯入流程進行機器可讀的比對。
	- 每筆記錄包含來源相對路徑、SHA-256、匯入日期、狀態與建立或更新的 My Wiki 頁面。
	- 這是流程管理文件，不是知識頁面，不需列入「Index.md」。

## 資料匯入與整理規則
- 請先掃描 /Row notes 資料夾，並讀取「Import Registry.md」判斷是否有需要匯入的新資料或已更新資料。
	- Registry 沒有該來源路徑：視為新資料，需要匯入。
	- 來源路徑相同且 SHA-256 相同：內容未變更，跳過匯入。
	- 來源路徑相同但 SHA-256 不同：視為來源已更新，重新檢查內容並更新既有頁面，避免建立重複頁面。
	- Row notes 是唯讀原始資料，不可為了標示匯入狀態而修改 Frontmatter、檔名、內容或所在位置。
- 接著讀取「Index.md」，了解目前已有哪些知識。
	- 如果新資料是關於已存在的知識或內容，請更新原本的頁面，不要重複建立新頁面。
	- 如果新資料內有出現新的重要知識或內容，才建立新頁面。	
- 在新增或更新知識內容時，專業的用詞請以英文為優先
- 在新增知識內容時，請優先採用 /Templates 資料夾中最符合內容用途的格式。
	- 每篇筆記只使用一個主要範本。
	- 建立筆記時，必須取代範本中的預留內容，並移除不適用的空白段落。
	- `type` 使用範本預設值；`status`、`topics`、`technology`、`language` 和 `source` 應依實際內容填寫。
	- `Related` 內的空白 `[[]]` 必須替換為至少一個實際相關頁面。
- 每個新頁面都要使用[[雙向連結]]至少連到一個相關頁面，避免變成孤立筆記。建立時請評估相關用詞
- 撰寫時請根據來源的語言來修改，但建立[[雙向連結]]時，請考慮評估使用既有的相關頁面，並將連結文字調整成合適內容。
	- 中文的內容請使用簡單流暢的白話寫作風格，不要使用文言文或艱深的詞彙。
- 每次資料整理完成後，你必須執行以下兩個任務
	- 更新「Index.md」以反映知識庫的最新內容。
	- 前往「Dialog.md」並添加一筆記錄。
	- 只有在 My Wiki、Index.md 與 Dialog.md 都成功更新後，才能在「Import Registry.md」新增或更新該來源的路徑、SHA-256、匯入日期、狀態與 Output pages。
	
## 回答提問的要點
- 當我向你提問時，請先根據「Index.md」找出相關頁面，再根據那些頁面的內容來回答。
- 在你的回覆中請清楚標明你引用了哪些頁面的資訊，讓我能知道答案的依據來源。
- 如果我們的問答中有值得保留的資訊，你可以建議我將這些資訊存到「知識庫」，但儲存前請先詢問我。

## 知識庫維護檢查要點
如果我要求你針對知識庫進行維護的話，請你進行以下的內容檢查：
- 檢查是否有失效的[[雙向連結]]。
- 檢查是否有孤立的頁面 (沒有被其他頁面連結到)。
- 檢查是否有重複或高度相似的頁面，並向我提出合併頁面的建議。
- 檢查頁面之間是否有內容的矛盾和不一致。
- 任務完成後請輸出一份簡短的報告。
