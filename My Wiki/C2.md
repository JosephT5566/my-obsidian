# C2

## Summary

C2 是 [[Houzz]] 的 legacy web application 與 backend service platform。它同時承接 server-rendered page、business logic、batch/database workflows，以及透過 [[Apache Thrift RPC]] 提供給 [[Jukwaa]]／GraphQL 使用的 service handlers。

## Architecture

兩條常見 request path 是：

```text
C2 page → C2 application/service → database
Jukwaa React → GraphQL resolver → Thrift client → C2 handler → database
```

[[C2-Jukwaa Web Module]] 會把兩個系統串在一起：C2 page 發出 Web Module request，由 Jukwaa handler 與 React component 渲染 UI；資料再經 GraphQL 與 Thrift 回到 C2 service。

## Development and Operations

- [[C2 Development on Kubernetes]]：透過 AWS SSO、EKS、Codepath pod 與 VS Code 修改 `c2web` 或 `c2thrift` container 中的程式。
- [[C2 Thrift Socket Troubleshooting]]：調查 development environment 的 Unix socket provisioning 與 endpoint configuration。
- [[Email Template Testing]]：在 staging RQ debug pod 驗證 C2 email templates。
- [[Batch Jobs and Database Migrations]]：測試 RQ jobs，並以版本化 SQL 與 AWX/Ansible 更新 schema。
- [[User Impersonation Safety]]：以最小風險重現特定使用者在 Houzz UI 的狀態。

## Shared Technologies

C2 的實作會使用 [[Kubernetes Architecture and Debugging|Kubernetes]] 作為開發或執行環境，以 [[Apache Thrift RPC|Thrift]] 維持跨語言 service contract，並可能透過 [[Redis Debugging|Redis]] 檢查 cache state。這些技術頁保留通用原理，C2 頁則保留 Houzz-specific usage。
